_divvy_ram(ram)¶

This is an arbitrary, somewhat meaningless method, to simulate the notion of free blocks of ram that must be allocated and referenced via pointer. A given program may require X ram, but the location of available ram may be disparate, so various blocks have to be stored as a free linked list, or similar data structure. Since we’ve already covered linked lists, association lists, etc..., there isn’t much value outside learning the context of this data structure, which tends to be in memory management.
